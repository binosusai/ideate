# Contributing to ideate

## Working Style

- Keep changes focused and presentation quality high.
- Prefer small, reviewable pull requests over large mixed changes.
- Preserve the investor-facing shell when improving the dashboard.

## Before Opening a Pull Request

- Run the relevant validation for the area you touched.
- Note user-visible changes clearly in the pull request description.
- Avoid unrelated refactors in the same contribution.

## Dashboard Contributions

- Keep the visual language intentional, elegant, and easy on the eyes.
- Treat motion as product storytelling, not decoration.
- Make sure desktop and mobile both read cleanly.

## Pull Requests

- Describe the problem, the change, and how you validated it.
- Include screenshots or short recordings for UI work when possible.
- Link any relevant issue, idea, or design context.

## Conduct

- Collaborate directly, respectfully, and with clear technical reasoning.
