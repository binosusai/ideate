## Why

Ideate currently has strong idea-level OpenSpec artifacts but no parent-level OpenSpec change that defines how project-wide spec-driven development should be organized and enforced. Adding this now creates a stable governance layer before further feature growth and paid-product hardening.

## What Changes

- Add a parent OpenSpec change under `ideate/openspec/changes/ideate-governance/`.
- Define parent-level requirements for structure, traceability, and validation of idea-level specs.
- Deliver concrete governance-linked runtime hardening and quality-loop features so the parent change stays implementation-grounded.
- Establish a roadmap of additional capabilities to improve spec quality, consistency checks, and release readiness.

## Capabilities

### New Capabilities
- `parent-openspec-governance`: Define parent-level OpenSpec requirements and workflows for ideate.

### Modified Capabilities
- `parent-openspec-governance`: Expanded from baseline process-only scope to include implemented, governance-linked runtime improvements.

## Impact

- Adds and updates parent-level OpenSpec artifacts.
- Includes implemented runtime and workflow changes in ideate CLI, data lifecycle validation, POC quality scoring, and feedback iteration workflows.
- Improves development process quality and project readiness for structured, paid-product evolution with measurable quality deltas.
